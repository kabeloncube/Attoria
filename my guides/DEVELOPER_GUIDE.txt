# 🚀 Developer Guide - CoC Dashboard

## 📁 File Structure & What Each File Does

```
coc-backend/
├── server.js          # Main backend server (Express + API routes)
├── .env               # Secret API key storage (NEVER commit to git)
├── package.json       # Project dependencies and scripts
├── users.db          # SQLite database file (auto-created)
├── public/           # Frontend files served to browser
│   ├── index.html    # Main webpage with styling
│   └── app.js        # Frontend JavaScript (handles UI interactions)
```

## ⚠️ CRITICAL - Don't Break These!

### **🔴 NEVER TOUCH THESE:**
1. `.env` file - Contains your API key
2. `users.db` - Your user database  
3. `node_modules/` folder - Installed packages

### **🟡 BE CAREFUL WITH:**
1. **API Endpoints** in server.js - URLs must match frontend calls
2. **HTML IDs** - JavaScript looks for specific element IDs
3. **Function Names** - Called from other parts of code

### **🟢 SAFE TO MODIFY:**
1. **CSS Styles** - Colors, fonts, spacing
2. **HTML Content** - Text, headings, descriptions
3. **Console.log()** statements - For debugging

## 🛠️ Common Edits You Might Want:

### **Change Colors:**
```css
/* In index.html <style> section */
.clan-card { background: #your-color; }
.role-leader { background: #ff6b6b; }  /* Leader = red */
```

### **Add New API Endpoint:**
```javascript
// In server.js, add after existing routes:
app.get('/api/your-new-endpoint', async (req, res) => {
  try {
    const data = await fetchCoCAPI('/your-coc-api-path');
    res.json(data);
  } catch (error) {
    res.status(500).json({ error: 'Failed', message: error.message });
  }
});
```

### **Modify Member Display:**
```javascript

```

## 🐛 Debugging Tips:

1. **Check Browser Console** - F12 -> Console tab
2. **Check Server Logs** - Terminal where npm start is running  
3. **Test API Directly** - `curl http://localhost:3000/api/health`

## 📚 Learning Resources:
- **Express.js**: Web server framework
- **SQLite**: Simple database  
- **JWT**: User authentication tokens
- **Fetch API**: Making HTTP requests from browser
- **DOM Manipulation**: Changing webpage content with JavaScript

## 🚨 Emergency Recovery:
If you break something, you can always:
1. Stop server: `pkill -f "node server.js"`
2. Restart: `npm start`
3. Check this guide for what each part does!
